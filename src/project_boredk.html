Project Introduction

The Bachelor Exam Project

This project was a part of my 3rd and final semester of my Web Development education. The assignment was to identify an issue within a company and design and develop a digital product that could work as a solution to the problem. Based on the product needs and size it was important to choose the right tech stack and where we could show our knowledge of choosing the right techstack. The product solution should show our techskills in both frontend and backend development as well as database modelling. Furthermore the project should include DevOps and Web Security implementations. This was the first time I developed an application in ReactJS and worked with the MySQL database.

Problem Formulation

How can we develop an intuitive and secure web application with a focus on software quality, which offers Danske Bank's internal employees a centralized platform for knowledge sharing and project management?

Product Solution

The new release hub, BOREDK Spotlight Platform, is primarily intended for internal project sharing and tracking deliveries from various squads or business teams within Danske Bank. The release hub should include an overview of the different squads, tribes and projects as well as the roles of employees, such as developers, product owners, business analysts, IT architects, etc. It should also feature a release pipeline and a changelog that provides an overview of the latest releases in each squad. Therefore, the application should function as a knowledge-sharing platform, allowing all employees at Danske Bank to stay updated on activities across different departments.

Highlighted projectelements

  • Login as regular employee or admin user and get redirected to different pages
  • Regular employee: Get overview of all tribes, squads and projects with relevant information and statuses. Like/favorize interesting projects and get personalized content or easy access to the data later
  • Admin user: See, create, update and delete all data in admin dashboard
  • Dockerization and CI-Pipeline
  • Testing: Unit tests, End-to-end tests (Cypress), API tests & Linting
  • Security: Authentication, authorization, sanitization, cookies, tokens, middleware, validation & hashing
